Unsupervised Learning on Country Data¶

A dataset of countries and various metrics (child mortality, GDP per person, ...) that indicate how "developed" they are. The goal is to identify the countries that are most in need of aid from an NGO.

From Kaggle: https://www.kaggle.com/datasets/rohan0301/unsupervised-learning-on-country-data/data

(The wording is yikes-inducing, it describes them as "backward" countries).

Here, we implement k-means clustering to divide the countries into groups. To determine the optimal number of clusters, k, we use a combination of the silhouette method and the elbow method. Furthermore, for each value of k, we run the clustering multiple times and pick the "best" clustering (according to the silhouette method), just in case there's an unlucky run due to the random starting centroids being bad.

Implementation of k-means clustering¶

In [196]:
class KMeans:
    def __init__(self, k):
        self.k = k
    
    def fit(self, X, max_rounds=100, starting_centers=None):
        minmaxes = list(zip(np.min(X, axis=0), np.max(X, axis=0)))
        if starting_centers is None:
            self.C = np.array([[random.uniform(minval, maxval)
                                for minval, maxval in minmaxes]
                               for _ in range(self.k)])
        else:
            self.C = starting_centers
        self.groups = np.zeros(len(X), dtype=np.uint)
        # Repeatedly assign datapoints to clusters and update the
        # centroids, until convergence.
        self.rounds = 0
        while self.rounds < max_rounds:
            num_changed = self._update_centroids(X)
            self.rounds += 1
            if num_changed == 0:
                break
        
        self.group_sizes = dict([(group_id, 0) for group_id in range(self.k)])
        unique, counts = np.unique(self.groups, return_counts=True)
        for group_id, count in zip(unique, counts):
            self.group_sizes[group_id] = count

    def _update_centroids(self, X):
        new_groups = np.zeros_like(self.groups)
        for i, x in enumerate(X):
            diffs = np.sum((self.C - x)**2, axis=1)
            new_groups[i] = np.argmin(diffs)
        num_changed = np.sum(self.groups != new_groups)
        self.groups[:] = new_groups
        if num_changed > 0:
            for group_idx in range(self.k):
                mask = self.groups == group_idx
                if np.sum(mask) != 0:
                    self.C[group_idx] = np.mean(X[mask], axis=0)
        return num_changed
    
    def compute_sq_dist_sum(self, X):
        """Once the clusters have been established via `fit(...)`, this metric
        gives the squared Euclidean distance from each point to the center of its cluster.
        """
        s = 0
        for i, c in enumerate(self.C):
            s += np.sum((X[self.groups == i] - c)**2)
        return s
    
    def silhouette_score(self, X):
        silhouette_score = 0
        for group_id in range(self.k):
            group_size = self.group_sizes[group_id]
            this_cluster = X[self.groups == group_id]
            for x in this_cluster:
                silhouette_score += self._calc_silhouette_score(x, X, this_cluster, group_id, group_size)
        return silhouette_score/len(X)
    
    # Mixing "cluster" and "group" terminology, woops.
    def _calc_silhouette_score(self, x, X, cluster, group_id, group_size):
        if group_size == 1:
            return 0
        # It's okay that we don't exclude x from the mask because the
        # distance from x to x is 0, obvs.
        sim_score = (1/(group_size-1)) * np.sum(np.sqrt(np.sum((x-cluster)**2, axis=1)))
        dissim_score = min(self._calc_dissim_score(x, X, j)
                           for j in range(self.k)
                           if j != group_id)
        return (dissim_score - sim_score)/max(sim_score, dissim_score)
    
    def _calc_dissim_score(self, x, X, j):
        cluster_size = self.group_sizes[j]
        if cluster_size == 0:
            return 0
        # Really need to extract the euclidean distance thing into a function.
        return np.sum(np.sqrt(np.sum((x-X[self.groups == j])**2, axis=1))) / cluster_size

Eyeballing the plot above, the "elbow" seems to be at k=3 or k=4, though the shape varies wildly between runs. So I think it depends a lot on the centres we start with. Let's try the silhouette method, and also do multiple runs for each k to hopefully mitigate the issue of bad starting centers.

    print("k:", k)

    score = -2
    C = None
    for _ in range(runs_per_k):
        model = KMeans(k)
        model.fit(X)
        new_score = model.silhouette_score(X)
        if new_score > score:
            C = model.C
            score = new_score
    best_Cs.append(C)
    scores.append(score)

Between the silhouette score and the elbow method, the ideal number of clusters seems to be k=3 or k=4. I suspect the reason the score plummets for k=6 is that we consistently get empty clusters, which tanks the score. This could be mitigated by initializing the cluster centers differently, e.g. sampling random points and choosing them as the starting centers, or sampling groups of points and taking their means.

In any case, let's see what the clustering looks like with k=3 and k=4, reusing the centers that we got from the parameter testing.

Conclusions¶

We've seen how the k-means algorithm can be applied to clustering of country data. The value of k that seems to best divide the countries into "haves" and "have-nots" is k=4. k=2 gave a similar divide, but it lumped relatively wealthy countries like Argentina into the "have-nots" category (according to my ignorant view of these countries, anyway). k=4 also adds two extra categories -- tax havens and Nigeria. In reality, however, defining a dividing line between poor and rich countries is somewhat blurry and arbitrary, as well as being a distinction that comes dangerously close to prejudice / white saviorism / other unpleasant things. So... this probably isn't the right approach to allocating funds for NGOs, as premised by the Kaggle problem.

Anyway, future work:

  • Look at individual countries and figure out what makes them distinct. Why does Nigeria end up in its own cluster? For example, it could have the unusual combination of being wealthy and having a high birth rate. The tax havens are probably distinguished by a high GDP per person. Lots of interesting countries to look at.
  • Implement alternative methods of picking the starting point for clusters.
